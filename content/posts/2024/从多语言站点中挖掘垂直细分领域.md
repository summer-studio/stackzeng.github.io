很多站点会做多语言，一套系统适配多种语言。

如果我们能复制站点到某个小众语言中，并用全站去优化，那么是否可以拿到垂直细分领域的流量呢？

## 数据验证

以 https://morsedecoder.com/ 为例说明，网站的功能非常简单，提供将字符转为摩斯码的功能。

网站每月的访问量大概 66 万左右。

![img](/images/2024/Snipaste_2024-08-29_16-37-46.png)

按照地理位置来源划分看下流量来源。

![img](/images/2024/Snipaste_2024-08-29_16-42-14.png)

土耳其占据每月访问流量的 6.39%，按照总访问量折算就是 42,000 左右。

打开 semrush 搜索土耳其的主要搜索关键词，其实 similarweb 也有按照地域筛选关键词的功能，但是这个功能对应的价格需要几十万一年.....

![img](/images/2024/Snipaste_2024-08-29_16-49-37.png)

主要搜索词是 Mors Alfabesi Çeviri。

打开 https://ahrefs.com/keyword-difficulty 输入 Mors Alfabesi Çeviri，同时国家选择土耳其，检查优化难度。

![img](/images/2024/Snipaste_2024-08-29_16-50-19.png)

发现这个词的优化难度很低，说明是可行的。

## ChatGPT 创建代码

将 morsedecoder.com 进行截图输入 ChatGPT 让它按照图片生成对应的 HTML 代码。

在 Github 上建立私有仓库上传刚生成的 HTML 代码。

## Cloudflare 创建反向代理

打开 cloudflare DNS 管理，创建两条 cname 记录，content 是固定的填写 vercel 的 dns cname.vercel-dns.com。

![img](/images/2024/Snipaste_2024-08-29_15-11-46.png)

SSL 选择 FULL 模式。

![img](/images/2024/Snipaste_2024-08-29_15-12-21.png)

## Vercel 部署

关于 Vercel 之前的[文章](https://mp.weixin.qq.com/s?__biz=MjM5MDg1NTk0OA==&mid=2247486466&idx=1&sn=f928aca55652fa2567abd5345dc4dd97&chksm=a6bf2a6191c8a3773898f6efb742a7f0c27251e8dfb97d9275e83130674447218b9f1d54e1ad&token=1256457969&lang=zh_CN#rd)介绍过。

Vercel 后台选择创建 Project。

![img](/images/2024/Snipaste_2024-08-29_11-30-40.png)

选择刚才在 Github 上创建的私有仓库。

![img](/images/2024/Snipaste_2024-08-29_11-31-37.png)

部署 Project，勾选 override，这是因为我用的是纯静态的 HTML 代码。

![img](/images/2024/Snipaste_2024-08-29_11-33-08.png)

部署成功。

![img](/images/2024/Snipaste_2024-08-29_11-35-56.png)

## 自定义域名

Vercel 后台选择 Domain 菜单，输入自己的域名。

![img](/images/2024/Snipaste_2024-08-29_14-20-33.png)

选择域名的跳转形式，直接选择第一个，不过这个形式有点问题和现有的域名形式有点不一样，不过关系不大后面可以修改。

一般 abc.com 是顶级域名，www.abc.com 是二级域名，默认是顶级域名跳转到二级域名，而 Vercel 是反过来的。

![img](/images/2024/Snipaste_2024-08-29_15-10-52.png)

配置完成之后，界面上显示打钩就证明配置成功。

## 使用 Vercel 的优势

这种方式的优势在于大量节约成本，不需要存储和带宽费用，成本只有域名的费用。

我可以复制大量的小众语种站点，哪个站点起来了就投入优化，不行就放弃，以数量换取成功的概率。
