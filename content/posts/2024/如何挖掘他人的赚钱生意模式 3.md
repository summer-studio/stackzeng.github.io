有很多网站会提供托管静态网站的服务，比如 gihub pages、vercel、netlify 等等。 

在这些网站上存在着大量开发者部署的项目和独立博客，且有着很多优秀的内容。

很多优秀的博客仅凭一篇文章就能从搜索引擎获得大量的、免费的流量。

如果我们能够找到这些散落在托管网站上的大量的优秀内容，再根据目标网页内容做一个网站，用全站的力量和单页竞争，截取其中的流量，那么是否就可以获取到巨大的流量呢？

下面以 netlify 为例详细说明。

Netlify 是一个强大的平台，用于构建、部署和托管静态网站和现代 web 应用程序。

在 Netlify 上部署完成后会生成一个类似 https://xxxx.netlify.app 的 URL。

首先打开 semrush 输入 netlify.app，左侧菜单选择流量分析菜单，再选择主要界面选项卡。

![img](/images/2024/Snipaste_2024-06-24_15-01-50.png)

输入完成后界面上就会筛选出 netlify 上主要的流量入口网页。

接下来就是顺着网页逐个点击查找的过程。

首先打开 https://oji.netlify.app/，这是一个日文网页，功能描述是：叔叔句子生成器。

![img](/images/2024/Snipaste_2024-06-24_15-03-12.png)

每点击一次按钮就会生成一个日文句子，功能非常简单，看下网页的访问量。

![img](/images/2024/Snipaste_2024-06-24_15-02-53.png)

每月访问量有十几万，大部分流量来自搜索引擎。

页面功能非常简单，做一个这样的网站难度基本为零。

再来看第二个网站 https://translator-valyrian.netlify.app/

这是一个将英语翻译成瓦雷利亚语言的网站。

瓦雷利亚语言是乔治·R·R·马丁所著的冰与火之歌小说中虚构的语言，我也是冰与火之歌小说的忠实粉丝。

![img](/images/2024/Snipaste_2024-06-24_15-07-17.png)

看下网页的访问量。

![img](/images/2024/Snipaste_2024-06-24_15-03-24.png)

可以看到网页的访问量和电视剧的播出时间是息息相关的，其它时间访问并不多。

96% 的流量来自于搜索引擎。

查看页面源代码后发现翻译功能是通过调用接口实现。

![img](/images/2024/Snipaste_2024-06-24_15-03-44.png)

顺着调用的接口我又发现了一个有趣的网站 https://funtranslations.com/

这是一个翻译影视剧中各类语言的网站，且各类语言都有现成的 API 调用。

![img](/images/2024/Snipaste_2024-06-24_15-04-25.png)

如此简单的网页有这么大的访问量，那么我们大可以建立多个翻译语言的网站来和这些只有单一的网页竞争，以此来截取流量。

总结下思路。

先查找出三方托管服务中大流量的入口网页，再查看网页满足的功能。

找出自己有能力复制的网页，直接建立网站用全站的力量和单个网页竞争，截取流量为自己所用。
